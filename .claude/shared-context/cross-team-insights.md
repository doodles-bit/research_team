### 보고서 구조/커뮤니케이션
- "말할 수 있는 것/없는 것/필요한 것" 3x3 구조를 모든 보고서 Appendix에 적용 (하이파이러시 제안)
- 벤치마크 프레이밍: "비정상"이 아니라 "업계 범위 내 상단/하단"으로 톤 조절 (하이파이러시 → 코퍼헤드)
- 스튜디오 커뮤니케이션: "~로그 필요합니다" 대신 "~를 분석하려 했으나 데이터 부재로 불가했습니다"로 전달 (하이파이러시 → 전체)

### 데이터 품질/기술
- string 타입 숫자 컬럼 CAST 습관화 — PalM(use_diamond), Copperhead(missioncompletiondurationminutes) 모두 해당
- 테스트 종료일 데이터 처리: 마지막 날 반일 운영 데이터를 이탈로 오해하지 않도록 가이드 필요 (팔모+코퍼헤드 공통)
- 플랫폼/국가 디멘전 네이밍: BI 테이블 레벨에서 통일 추진 (크로스 프로젝트 비교 대비)

### 검증 프로세스 강화 (검증 제안, 전체 적용)
- 수치검증 + 서술검증 분리 운영
- 핵심(결론 인용) / 보조(본문 상세) / 파생(비율, 합산) 수치 태깅 → 핵심 전수, 나머지 샘플링
- 기준점 정의 일치 여부를 체크리스트 최상단에 (하이파이러시 → 검증)
- 합산 유저수 UNION vs UNION ALL 확인 항목 추가 (팔모 → 검증)
- 예방적 검증: 분석 착수 전 "이 분석의 흔한 오류 패턴" 가이드 사전 전달

### 비교 분석 시 통계적 유의성 검정 필수 (2026-03-31 사용자 피드백)
- 두 그룹 이상의 비율/수치를 비교할 때 반드시 95% CL 검정 수행 후 리포트에 포함
- 검정 방법: Wilson 95% CI + Two-proportion z-test (p-value 명시)
- 유의하지 않은 차이를 의미 있는 것처럼 서술 금지
- 사례: PalM Lv15 자발적 이탈 60%가 대조군 비교 시 Lv15 특유 현상이 아니라 레벨-이탈 시점 간 구조적 패턴으로 밝혀짐

- dungeon_level 컬럼: 숫자가 아니라 "Normal"/"Hard" 문자열. CAST(dungeon_level AS INT) 시도 시 에러 발생. → string 타입 숫자 컬럼 CAST 습관화와 별개로, 문자열 enum 컬럼도 있으므로 DESCRIBE 후 확인 필수.

### [2026-04-10] 데이터 시각화 가이드 공유 (hifirush-analyst → 전체 팀)

시각화 실무 가이드를 `shared-context/study-notes/data-visualization-guide.md`에 작성함. 전체 팀 참조용.

**모든 팀원 즉시 적용 권장 3건:**
1. **서술적 제목**: 차트 제목에 인사이트를 쓴다. "난이도별 완주율" (X) → "Master 완주율이 Normal의 2배" (O)
2. **강조 색상만 사용**: 핵심 항목 1~2개만 색상, 나머지는 회색. 전체에 다른 색 쓰면 강조 효과 소멸.
3. **변곡점 주석**: 라인/퍼널 차트에서 꺾이는 지점에 이벤트/원인을 짧은 주석으로 표기. 독자가 "왜?"라고 질문하기 전에 답 제공.

**안티패턴 (전체 팀 금지):** 3D 차트, 이중 Y축, 잘린 Y축(막대), 5+항목 파이, 무지개 색상
**색맹 안전 팔레트:** ColorBrewer (colorbrewer2.org) 사용 권장. 빨강-초록 대신 파랑-주황 조합.
