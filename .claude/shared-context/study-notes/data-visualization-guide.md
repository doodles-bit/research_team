## 1. 차트 유형별 적합한 상황

### 1-1. 차트 선택 의사결정 트리

**"무엇을 보여주고 싶은가?"에서 출발한다.**

| 메시지 유형 | 적합한 차트 | 부적합한 차트 |
|-------------|------------|--------------|
| **비교** (A vs B vs C) | 세로 막대, 가로 막대, 그룹 막대 | 파이, 도넛 |
| **추세** (시간에 따른 변화) | 라인, 영역(area) | 파이, 산점도 |
| **구성** (전체 대비 부분 비율) | 스택 바, 100% 스택 바, 트리맵 | 라인, 산점도 |
| **분포** (값의 퍼짐) | 히스토그램, 박스플롯, 바이올린 | 파이, 막대 |
| **관계** (두 변수 간 상관) | 산점도, 버블 차트 | 막대, 파이 |
| **흐름** (단계별 전환/이탈) | 퍼널, 산키(Sankey), 워터폴 | 라인, 파이 |
| **순위** (크기 순서 비교) | 가로 막대(정렬), 로리팝 | 세로 막대(비정렬), 파이 |
| **공간** (지역별 차이) | 히트맵, 코로플레스 맵 | 막대 (항목이 많을 때) |

> **출처:** Stephanie Evergreen, Effective Data Visualization (2nd ed., SAGE, 2019); Abela's Chart Chooser Diagram (extremepresentation.com)

### 1-2. 주요 차트 유형 상세 가이드

#### 세로 막대 (Vertical Bar)
- **언제:** 카테고리 간 크기 비교 (항목 수 7개 이하)
- **게임 사례:** 난이도별 완주율 비교 (Normal / Hard / Master)
- **주의:** 카테고리가 8개 이상이면 가로 막대로 전환. Y축은 반드시 0부터 시작.
- **변형:** 그룹 막대(2~3개 시리즈 비교), 스택 막대(구성 비교)

#### 가로 막대 (Horizontal Bar)
- **언제:** 카테고리가 많거나 라벨이 긴 경우, 순위 표현
- **게임 사례:** 국가별 유저 비중 (12개국 정렬), 아이템별 구매율 순위
- **주의:** 반드시 값 크기로 정렬 (알파벳 정렬 금지 -- 정렬이 메시지)
- **팁:** 강조할 항목만 색상 구분, 나머지는 동일 회색

#### 라인 (Line)
- **언제:** 시간 축이 있는 연속 데이터의 추세
- **게임 사례:** 스테이지별 잔류율 추이 (퍼널 라인), DAU 일별 추세
- **주의:** 시간 축이 아닌 카테고리 데이터에 라인 사용 금지 (연결 의미 없는 점을 이으면 오해 유발). 시리즈 3개 이하 유지.
- **팁:** 꺾이는 변곡점에 주석(annotation) 추가 = 인사이트 즉시 전달

#### 스택 바 / 100% 스택 바 (Stacked Bar)
- **언제:** 전체 대비 부분의 구성 비교
- **게임 사례:** 스테이지별 플레이스타일 비중 변화 (공격형/방어형/균형형)
- **스택 바:** 전체 크기도 비교 가능 + 구성도 표현
- **100% 스택 바:** 구성 비율만 비교 (전체 크기 차이 제거)
- **주의:** 스택 항목 4개 이하 유지. 중간 스택은 비교가 어려움 -- 가장 중요한 항목을 바닥(baseline) 또는 꼭대기에 배치.

#### 산점도 (Scatter)
- **언제:** 두 연속 변수 간 관계/상관 탐색
- **게임 사례:** 세션 길이 vs 결제 금액, 레벨 vs 이탈 확률
- **주의:** 점이 겹치면 투명도(alpha) 조절 또는 밀도 히트맵 전환. 상관관계 라인 추가 시 인과로 오해하지 않도록 주석 필수.

#### 히트맵 (Heatmap)
- **언제:** 두 개의 카테고리 축에 걸친 값의 강도/패턴 파악
- **게임 사례:** 스테이지 x 난이도 이탈률 히트맵, 시간대 x 요일 접속 패턴
- **주의:** 색상 스케일의 양 끝이 명확해야 함. 색맹 안전 팔레트 사용.
- **팁:** 셀에 수치 텍스트 표기 시 정밀한 비교 가능

#### 워터폴 (Waterfall)
- **언제:** 단계별 증감의 누적 효과 (시작 → 증가 → 감소 → 최종)
- **게임 사례:** 퍼널 이탈 시각화 (시작 유저 → T01 이탈 → T02 이탈 → ... → 완주 유저), 수익 구성 분해
- **주의:** 증가(양수)와 감소(음수)의 색상을 명확히 구분. 총계 막대 별도 표시.

#### 파이 / 도넛 (Pie / Donut)
- **언제:** 전체 대비 부분의 비율을 직관적으로 보여줄 때 (항목 2~3개일 때만)
- **게임 사례:** 플랫폼 점유율 (Steam 30% / PS5 20% / Xbox 50%)
- **경고:** 항목이 4개 이상이면 비율 비교가 어려움 -- 가로 막대로 대체 권장. 3D 파이 절대 금지.
- **출처:** Stephen Few, "Save the Pies for Dessert" (perceptualedge.com, 2007)

#### 퍼널 (Funnel)
- **언제:** 순차적 단계의 전환율/이탈 시각화
- **게임 사례:** 온보딩 퍼널 (설치 → 튜토리얼 → T01 → T02 → ... → 완주)
- **팁:** 각 단계 사이에 전환율(%) 또는 이탈률(%) 표기. 최대 병목 구간 강조 색상.

### 1-3. 게임 분석에서 자주 쓰이는 차트 조합

| 분석 유형 | 추천 차트 조합 | 우리 팀 사례 |
|-----------|---------------|-------------|
| 퍼널 이탈 분석 | 퍼널 차트 + 라인(잔류율) + 가로 막대(병목 순위) | Track 2 이탈 분석 |
| 리텐션 분석 | 라인(D1/D7/D30 추세) + 히트맵(코호트 리텐션 테이블) | 향후 PalM FGT |
| 플레이스타일 추이 | 100% 스택 바(스테이지별 비중) + 라인(특정 스킬 추세) | Track 1 플레이스타일 전환 |
| 아이템/경제 분석 | 가로 막대(구매율 순위) + 스택 바(카테고리별 구성) | Track 3 아이템 구매 |
| 플랫폼/국가 비교 | 그룹 막대(플랫폼별 KPI) + 히트맵(국가x지표) | Track 4, Track 5 |
| KPI 대시보드 | 스파크라인(추세) + KPI 카드(핵심 수치) + 막대(비교) | Copperhead 대시보드 |

## 2. 비전문가 독자를 위한 차트 디자인 원칙

### 2-1. 핵심 원칙: "차트를 보면 결론이 보여야 한다"

비전문가 독자(사업팀, 스튜디오)는 차트를 해독(decode)하는 데 시간을 쓰지 않는다.
차트가 메시지를 직접 전달해야 한다. 독자가 해석해야 하는 차트는 실패한 차트다.

> "Every chart must have a clear takeaway that the reader can articulate in one sentence."
> -- Cole Nussbaumer Knaflic, Storytelling with Data (Wiley, 2015)

### 2-2. 6가지 디자인 원칙

#### 원칙 1: 제목에 인사이트를 쓴다 (서술적 제목)

| 나쁜 예 | 좋은 예 |
|---------|---------|
| "난이도별 완주율" | "Master 유저 완주율이 Normal의 2배 -- 자기선택 편향 가능성" |
| "스테이지별 이탈률" | "T01 -> T02에서 절반 이상 이탈, 이후 안정화" |
| "국가별 유저 분포" | "US+CN 점유율 감소 추세, 다국화 진행 중" |

**규칙:** 차트 제목 = "무엇이 보이는가" (인사이트), 부제 = "무엇의 차트인가" (데이터 설명)

> **출처:** Knaflic (2015) ch.4 "Focus your audience's attention"; Evergreen (2019) ch.3 "Titles"

#### 원칙 2: 한 차트에 하나의 메시지

- 두 가지 메시지를 하나의 차트에 담으면 둘 다 약해짐
- 복잡한 내용은 차트 2개로 분리하는 것이 명확
- 예: "스테이지별 이탈률 + 플레이스타일 비중"을 하나에 담지 말고 각각 별도 차트

#### 원칙 3: 정보 과부하를 피한다 (잉크 대비 데이터 비율)

Edward Tufte의 "Data-Ink Ratio" 원칙:
- **제거 대상:** 3D 효과, 그라데이션 배경, 불필요한 격자선, 과도한 범례, 장식용 그래픽
- **유지 대상:** 데이터 포인트, 축 라벨, 단위, 출처 표기
- **판단 기준:** "이 요소를 제거하면 정보가 손실되는가?" -- 아니면 제거

> **출처:** Edward Tufte, The Visual Display of Quantitative Information (2nd ed., Graphics Press, 2001)

#### 원칙 4: 사전 주의 속성(Pre-attentive Attributes)으로 시선 유도

사람의 시각 시스템은 색상, 크기, 위치, 방향의 차이를 무의식적으로 포착한다.
이를 활용하여 핵심 데이터를 강조한다.

| 속성 | 활용법 | 예시 |
|------|--------|------|
| **색상 강도** | 핵심 항목만 강한 색, 나머지 회색 | 최대 병목 구간만 빨간색, 나머지 회색 |
| **크기** | 핵심 수치를 크게 표시 | KPI 카드의 핵심 숫자 폰트 크기 2배 |
| **위치** | 중요한 것을 왼쪽 상단에 배치 | Executive Summary의 핵심 차트를 첫 번째 위치 |
| **테두리/굵기** | 라인 차트에서 핵심 시리즈만 굵게 | 전체 평균 라인 굵게, 개별 세그먼트 얇게 |

> **출처:** Knaflic (2015) ch.3 "Clutter is your enemy"; Few, Now You See It (Analytics Press, 2009)

#### 원칙 5: 주석(Annotation)으로 맥락을 제공한다

- 변곡점, 이상치, 주요 이벤트에 텍스트 주석 추가
- 독자가 "왜 여기서 꺾이지?"라고 질문하기 전에 답을 제공
- 게임 사례: "T04에서 Macaron 합류 -- Partner 사용 급등", "T08에서 Korsica 합류"

**주석 규칙:**
- 화살표 또는 점선으로 데이터 포인트에 연결
- 간결한 문장 (10단어 이내)
- 차트 영역 내에 배치 (차트 밖 각주 X)

#### 원칙 6: 일관성 (Consistency)

- 동일 보고서 내에서 동일 엔터티는 동일 색상
  - 예: Steam = 항상 파란색, PS5 = 항상 보라색, Xbox = 항상 초록색
- 동일 보고서 내에서 동일 차트 유형은 동일 레이아웃
- 축 스케일을 차트 간 통일 (비교를 위해)
- 폰트, 크기, 정렬 일관성 유지

### 2-3. 텍스트 요소 활용법

| 요소 | 역할 | 규칙 |
|------|------|------|
| **차트 제목** | 인사이트 전달 | 서술형, "~이 ~보다 높음" 또는 "~추세 확인" |
| **부제** | 데이터 범위/조건 | "2025년 1월~12월, Steam 유저 기준" |
| **축 라벨** | 측정 항목 + 단위 | "완주율 (%)", "유저 수 (명)" -- 단위 필수 |
| **데이터 라벨** | 정확한 수치 | 막대 차트 끝에 수치 표기 (비교 시 필수) |
| **주석** | 맥락/이벤트 | 변곡점에 짧은 텍스트 |
| **출처** | 신뢰도 | 차트 하단 작은 글씨: "출처: stage_completion.csv, 전체 유저" |
| **범례** | 시리즈 구분 | 가능하면 범례 대신 직접 라벨(direct labeling) 사용 |

## 3. 색상, 레이블, 레이아웃 실무 가이드

### 3-1. 색상 팔레트 선택

#### 색상의 3가지 용도

| 용도 | 설명 | 팔레트 유형 | 예시 |
|------|------|------------|------|
| **범주 구분** | 서로 다른 카테고리 | 정성적(Qualitative) | 플랫폼별: Steam=파랑, PS5=보라, Xbox=초록 |
| **크기/순서** | 값의 크고 작음 | 순차적(Sequential) | 이탈률 낮음=연한 파랑 → 높음=진한 파랑 |
| **양극** | 기준 대비 양/음 | 발산적(Diverging) | 평균 대비 증가=파랑, 감소=빨강 |

#### 실무 색상 규칙

1. **핵심 1~2개만 강조색, 나머지는 회색**
   - 모든 항목에 다른 색을 쓰면 강조 효과가 사라짐
   - "전략적 색 사용(strategic use of color)" = 가장 중요한 것만 색을 입힘
   - > 출처: Knaflic (2015) ch.4

2. **빨강 = 부정/경고, 파랑 = 중립/긍정, 초록 = 긍정/달성**
   - 문화적 관습을 따름. 독자가 추가 해석 없이 의미 파악 가능.
   - 단, 빨강-초록 조합은 색맹(적녹색맹 ~8% 남성)에 불리 -- 파랑-주황 조합이 안전.

3. **색맹 안전 팔레트 사용**
   - 추천: ColorBrewer 팔레트 (colorbrewer2.org) -- 학술/업무 표준
   - 대안: Tableau 기본 팔레트, Viridis 계열 (연속형에 적합)
   - > 출처: Cynthia Brewer, ColorBrewer 2.0 (colorbrewer2.org)

4. **동일 보고서 내 색상 일관성**
   - 한번 정한 엔터티-색상 매핑은 보고서 전체에서 유지
   - 팀 표준 색상 매핑 예시:
     - Steam: #1B9AAA (청록)
     - PS5: #6C5CE7 (보라)
     - Xbox/Other: #A8A8A8 (회색) -- 주요 분석 대상이 아닐 때
     - 강조/경고: #E74C3C (빨강)
     - 기준선/평균: #95A5A6 (연회색, 점선)

#### 게임 분석 색상 팁
- 난이도 순서가 있으면 순차적 팔레트: Normal=연한색 → Master=진한색
- 이탈/잔류 구분: 이탈=빨강 계열, 잔류=파랑 계열
- 플레이스타일 구분: 정성적 팔레트 (서로 확실히 구분되는 색상)

### 3-2. 축, 범례, 격자선 정리

#### 축(Axis) 규칙

| 규칙 | 이유 | 예외 |
|------|------|------|
| Y축은 0부터 시작 | 막대 차트에서 높이 비율이 왜곡됨 | 라인 차트에서 변동폭 강조 시 예외 가능 (명시 필수) |
| 축 라벨에 단위 포함 | "25"만 있으면 25명인지 25%인지 모름 | -- |
| 불필요한 눈금(tick) 제거 | 시각적 잡음 감소 | 정밀 비교 필요 시 유지 |
| 시간 축은 등간격 | 1월-3월-12월 처럼 불균등하면 추세 왜곡 | -- |

#### 범례(Legend) 규칙

- **가능하면 직접 라벨링(Direct Labeling)** 사용 -- 라인 끝에 시리즈명 표기
  - 이유: 독자가 범례와 차트를 왔다갔다 하는 인지 비용 제거
- 범례를 쓸 때: 차트 상단 또는 우측에 배치 (하단은 잘려 나갈 위험)
- 범례 항목 순서 = 차트 내 시리즈 순서와 일치
- > 출처: Knaflic (2015) ch.3; Few, Show Me the Numbers (2nd ed., Analytics Press, 2012)

#### 격자선(Gridline) 규칙

- 주 격자선: 연한 회색, 가능하면 점선
- 보조 격자선: 대부분의 경우 제거
- 배경색: 흰색(투명). 회색/컬러 배경 사용 금지.
- "격자선은 있어도 안 보일 정도로 연해야 한다"

### 3-3. 차트 크기와 비율

- **황금비 근사 (약 1.6:1)** 가 일반적으로 가독성이 좋음
- 라인 차트: 가로로 넓게 (추세가 완만해 보여야 과장 방지)
- 막대 차트: 막대 사이 간격 = 막대 너비의 50~100%
- 스몰 멀티플(Small Multiples): 동일 차트를 조건별로 반복 배치 -- 3x2 또는 4x2 그리드. 큰 차트 하나보다 작은 차트 여러 개가 비교에 유리한 경우 있음.
- > 출처: Tufte (2001) ch.8 "Small Multiples"

### 3-4. 흔한 실수와 안티패턴

#### 절대 하지 말 것 (안티패턴)

| 안티패턴 | 왜 나쁜가 | 대안 |
|---------|----------|------|
| **3D 차트** | 원근법으로 크기 비교 불가. 뒤쪽 막대가 가려짐. 장식적이지만 정보 전달을 방해. | 2D 차트 사용 |
| **이중 Y축** | 두 축의 스케일에 따라 상관관계가 있는 것처럼 보이거나 없어 보임. 조작 가능. | 차트 2개로 분리 (동일 X축 공유) |
| **잘린 Y축 (막대 차트)** | 막대 높이가 실제 비율을 왜곡. 5%와 10% 차이가 2배로 보이지 않음. | Y축 0부터 시작. 또는 차이를 별도 차트로 표시. |
| **파이 차트에 5개 이상 항목** | 인간의 각도 비교 능력 한계. 비슷한 크기 구분 불가. | 가로 막대 사용 |
| **무지개 색상** | 순서 정보 없음, 색맹 비우호적, 시각적 혼란 | 단색 순차적 팔레트 또는 정성적 팔레트 |
| **라벨 없는 축** | 무엇을 보는지 모름 | 축 라벨 + 단위 필수 |
| **정렬 안 된 막대** | 순위 파악 어려움, 비교 비효율적 | 값 크기순 정렬 (시간 축 제외) |
| **범례만 있고 직접 라벨 없음** | 독자가 색상-항목 매칭을 반복적으로 확인해야 함 | 직접 라벨링 우선 |

> **출처:** Tufte (2001); Few, "Save the Pies for Dessert" (perceptualedge.com, 2007); Knaflic (2015) ch.3

#### 주의해서 쓸 것

| 차트/기법 | 주의 사항 |
|-----------|----------|
| **영역 차트 (Area)** | 겹침으로 뒤 시리즈가 가려질 수 있음. 스택 영역으로 사용하거나 시리즈 2개 이하. |
| **버블 차트** | 면적 비교는 직관적이지 않음. 크기 범례 필수. |
| **트리맵** | 세부 항목이 많으면 라벨 가독성 저하. 최상위 카테고리만 표시. |
| **산키 다이어그램** | 복잡도가 올라가면 스파게티화. 5개 이하 노드 권장. |

## 4. 우리 팀 실무 적용 체크리스트

### 차트 작성 전 체크리스트

- [ ] **메시지 먼저**: "이 차트로 독자에게 전달할 한 문장이 무엇인가?" 정의
- [ ] **차트 유형 선택**: 메시지 유형(비교/추세/구성/분포/관계/흐름)에 맞는 차트 선택 (1-1 참조)
- [ ] **데이터 준비**: 차트에 필요한 데이터만 추출. 불필요한 카테고리 제거 또는 "기타"로 묶기

### 차트 작성 중 체크리스트

- [ ] **서술적 제목**: 제목이 인사이트를 담고 있는가 (2-2 원칙 1)
- [ ] **부제 + 출처**: 데이터 범위와 출처 표기
- [ ] **축 라벨 + 단위**: 양쪽 축에 라벨과 단위 포함
- [ ] **Y축 0 시작** (막대 차트): Y축이 0부터 시작하는가
- [ ] **색상**: 강조 항목만 색, 나머지 회색인가. 색맹 안전 팔레트인가.
- [ ] **데이터 라벨**: 비교가 필요한 막대에 수치 표기
- [ ] **주석**: 변곡점/이상치에 설명 주석이 있는가
- [ ] **범례 vs 직접 라벨**: 가능하면 직접 라벨링 사용
- [ ] **격자선**: 연한 회색, 최소한으로 유지
- [ ] **잉크 대비 데이터 비율**: 제거해도 정보 손실 없는 요소가 있는가 -> 제거

### 차트 완성 후 체크리스트

- [ ] **1-문장 테스트**: 차트를 3초 보고 한 문장으로 메시지를 말할 수 있는가
- [ ] **비전문가 테스트**: 데이터를 모르는 사람이 봐도 이해할 수 있는가
- [ ] **일관성**: 같은 보고서 내 다른 차트와 색상/스케일/폰트가 일관적인가
- [ ] **안티패턴 검증**: 3D, 이중 Y축, 잘린 축, 무지개 색상 등 사용하지 않았는가
- [ ] **출처 표기**: 데이터 출처(테이블명, 필터 조건)가 차트 하단에 있는가
