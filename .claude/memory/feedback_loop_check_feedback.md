description: 자율 loop 시작 시 반드시 feedback/ 디렉토리를 확인하여 늦게 도착한 외부 피드백을 처리
type: feedback
---

자율 loop에서 Phase 5(외부 피드백)를 "사용자 부재"로 일괄 스킵하면 안 된다. 피드백은 비동기로 늦게 도착할 수 있다.

**Why:** 분석 팀장이나 사용자가 리포트를 읽고 feedback/ 에 파일을 작성하는 데 시간이 걸릴 수 있음. loop가 피드백 확인 없이 새 주제만 진행하면 피드백이 영원히 반영되지 않음.

**How to apply:**
- 매 loop 시작 시 첫 단계로 `feedback/` 디렉토리에 신규/미처리 파일이 있는지 확인
- 신규 피드백이 있으면 해당 리포트 수정을 새 연구보다 우선 처리
- 처리 완료 후 피드백 파일에 "반영 완료" 표기 또는 팀장 메모리에 기록
- 피드백이 없을 때만 새 연구 주제로 진행
