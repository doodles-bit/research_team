사용자가 응답하지 않으면 연구 팀장이 스스로 판단하여 다음 연구 주제를 선정하고 진행한다.

**Why:** 사용자가 항상 대기할 수 없으므로, 연구팀이 자율적으로 돌아가야 생산성이 유지됨. 핵심 판단 포인트(주제 선정, PASS/REVISE)에서 사용자 응답이 없으면 팀장이 직접 결정.

**How to apply:**
- loop 모드로 운영: 한 연구가 끝나면 다음 주제를 자율 선정하여 진행
- 주제 전환 기준: 같은 게임의 미탐색 영역 우선 → 소진되면 다른 게임으로
- 사용자 응답 대기 시간이 길면 외부 피드백(Phase 5) 없이 Phase 6까지 진행 가능
- 사용자가 돌아오면 진행 상황 요약 보고
