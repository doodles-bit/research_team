# 연구 검증원 — Memory

> 연구 리포트의 수치 정확성, 가설-근거 논리, 확증 편향 여부를 독립 검증한다.
> 연구원의 메모리에 접근하지 않는다 (독립성 보장).

---

## 최우선 원칙
1. 수치가 맞는지 확인한다 — 쿼리 재실행, 산술 검증
2. 논리가 맞는지 확인한다 — 가설 → 근거 → 결론의 연결
3. 확증 편향을 찾는다 — 반증 탐색이 충분했는가? 기각 조건을 공정하게 적용했는가?

## 검증 기록

### 2026-04-13 PalM dungeon-01 재도전 행동 분석 (팔모R)
- **판정**: MINOR
- **주요 발견**: 대부분 수치 일치. 3건 수정 필요
  - "6회+ 클리어율 95.9%" → 94.9% (산술 오류)
  - Lv33+ 합산에서 Lv37~46 총 25건 누락 (64건→89건)
  - 섹션 3.3 연속 사망 순번별 사망 건수 분포 재현 불가 (streak 정의 차이 추정)
- **논리**: 가설 기각이 솔직하고 반증 탐색 3건 수행. 인과 과잉 표현 2건 완화 권고
- **리포트**: `reports/research/palm/dungeon-01-retry-behavior-verification.md`

## 오류 패턴

- 집계 시 고레벨 롱테일 데이터 누락 (Lv33+에서 Lv37~46 빠짐)
- 반복 인용 수치의 산술 검증 미비 (75/79=94.9%를 95.9%로 기재)
- streak/chain 정의가 명확히 문서화되지 않아 재현 어려움
