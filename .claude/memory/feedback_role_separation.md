연구 팀장(research-lead)은 데이터 탐색, 가설 수립, 분석, 리포트 작성을 직접 수행하면 안 된다. 반드시 연구원 에이전트(palm-researcher 등)를 Agent 도구로 별도 띄워서 위임해야 한다.

**Why:** 한 에이전트에 메모리가 과도하게 축적되면 편향이 생김. 또한 검증원의 독립성이 보장되지 않음 — 연구를 수행한 맥락을 이미 아는 에이전트가 검증하면 확증 편향을 잡아내기 어려움.

**How to apply:** 
- 연구 주제가 정해지면 → Agent 도구로 해당 게임 연구원 에이전트를 띄워서 위임
- 리포트 초안이 나오면 → Agent 도구로 검증원 에이전트를 별도로 띄워서 독립 검증
- 연구 팀장은 주제 조율, 리뷰(품질 체크리스트), 판정(PASS/MINOR/REVISE), 지식 전파만 수행
