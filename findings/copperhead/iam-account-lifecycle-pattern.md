## IAM 계정 등록·로그인 생명주기

- **인사이트**: Copperhead 베타의 44,549개 계정이 100% headless(기기 자동 인증) 방식으로 등록·로그인하며, 등록-로그인이 1초 이내 단일 단계로 처리된다. 2025-11-28~12-13(16일) 동안 IAM 이벤트가 정확히 2배로 중복 기록되는 텔레메트리 버그가 존재하며(1,151개 등록 / 1,510개 로그인 계정 영향), 토큰 갱신은 전체 2.9% 계정에서만 발생하나 이 계정의 로비 접속이 비갱신 대비 3.9배로 장시간 활동의 지표다.
- **근거**:
  - `view_iam_registered`, `view_iam_logged_in`, `view_client_gpp_token_refresh` 전수 분석
  - 등록-로그인 시간차: 99.99%(44,543/44,549건) 1초 미만, 기기 일치율 100%
  - 이중 기록: 해당 기간 이벤트/고유계정 비율 정확히 2.00, 밀리초까지 동일한 완전 중복
  - 토큰 갱신 계정(1,286명): 로비 접속 4.08회 vs 비갱신 1.05회(+289%), 첫 갱신까지 ~48분(2,879초)
  - 자동화 기기에서 토큰 갱신 0건 — 자동화 세션은 48분 이내 종료 추정
  - 전체 퍼널: 등록 100% → 로비 98.8% → 세션 72.1%
- **활용 제안**:
  - **이중 기록 기간(11/28~12/13) 데이터 분석 시 중복 제거 필수** — IAM 기반 집계에 직접 영향
  - 토큰 갱신 여부를 장시간 세션 유지 계정 식별 지표로 활용 가능
  - 로비→세션 27.9% 이탈 구간이 주요 전환 개선 포인트
  - 공식 자동화 7대 외 비공식 대량 계정 생성 기기(5대, 1,000+계정) 존재 가능성
