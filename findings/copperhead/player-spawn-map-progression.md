# Copperhead 맵 진행 퍼널 — 37.4% 로비 이탈, 허브-스포크 맵 구조, 리스폰≠난이도

## 인사이트
Copperhead 베타 스폰 데이터 1,795명(자동화 제외) 중 37.4%(671명)가 로비에서 한 번도 미션 맵에 진입하지 않았으며, 이 중 69.7%(468명)는 단 한 번 스폰 후 기록이 종료됐다. 미션 맵은 번호 순서대로 순차 진행되지 않고, 로비를 중심으로 한 독립 선택(허브-스포크) 구조이다. 미션 내 리스폰 횟수는 성공률과 양의 상관을 보이나, 이는 생존자 편향(성공 미션이 더 오래 진행→리스폰 누적)이다.

## 근거
- 6,599건 스폰, 1,795명, 21개 맵 분석 (`cphplayerspawned`, 2026-02-04~, 자동화 필터링 적용)
- 로비-only 671명(37.4%), 미션 1개만 901명(50.2%), 2개+ 103명(5.7%)
- 맵 전환: 모든 전환이 로비 경유. MIS_04(128명) > MIS_03(87명) — 번호 순서와 무관
- MIS_01 미경험 상태로 MIS_02~04 직행한 플레이어 264명(미션 방문자의 26.3%)
- MIS_01 리스폰별 성공률: 0회 10.6% → 4+회 55.4% (생존자 편향, 이전 연구 #4와 일관)
- 단, 스폰 텔레메트리 커버리지 40.1%(비자동화 4,480명 중 1,795명만)

## 활용 제안
- **로비 이탈 원인 조사**: 37.4%가 미션 미진입 — 크래시/로딩 실패인지 UX 가이드 부족인지 구분 필요. `view_lobby_disconnected` 테이블 활용 가능
- **맵 선택 UI 검토**: 맵 번호가 순서/난이도를 의미하지 않으므로, 플레이어에게 맵 특성(난이도, 콘텐츠)을 명확히 전달하는 UI 검토
- **리스폰을 난이도 지표로 사용 주의**: 리스폰 빈도는 미션 시간과 교란되므로, 밸런싱 근거로 사용 시 단위 시간당 리스폰(리스폰/분)으로 보정 필요
