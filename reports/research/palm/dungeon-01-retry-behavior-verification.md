#### "6회+ 시도자 클리어율 95.9%" 수치 오류

리포트 요약 및 본문에서 반복 인용되는 "6회 이상 시도자 클리어율 95.9%"는 **산술 오류**.

- 6~10회(17/21) + 11~20회(38/38) + 21+(20/20) = **75/79 = 94.9%**
- 리포트의 95.9%는 출처 불명. 76/79 = 96.2%, 75/79 = 94.9%이므로 정확한 값은 **94.9%**

> 요약(3번 항목), 섹션 3.4 본문, 섹션 5.2 비교표에서 모두 95.9%로 기재되어 있어 일괄 수정 필요.

## 논리 검토

### 가설 정의

- 가설이 명확히 서술됨: "연속 실패 시 재도전율이 감소하며, 3~5회에서 포기 분기점이 존재"
- 예상 결과가 정량적으로 제시됨 (1~2회: 50%+, 3~5회: 20~30%, 6+: 10% 미만)
- 기각 조건이 명시됨 (감소 관계 없으면 기각, 유지/상승이면 기각)
- **평가: 양호** — 사전에 가설·예상·기각조건이 정의된 구조

### 가설 → 근거 → 결론 흐름

1. **가설 기각 판정**: 섹션 3.3에서 연속 사망 순번이 올라가도 재도전율이 감소하지 않고 오히려 상승하므로 기각. 이 논리는 타당함
2. **수정 발견 ("건강한 벽")**: 해석 A(좌절) vs 해석 B(도전)의 프레임을 제시하고, 5개 지표에서 모두 B가 우세함을 보여줌. 비교표(섹션 5.2)가 명확하여 논리 추적이 용이
3. **인과관계 제한**: "유저가 즐거워하는지는 알 수 없다"를 반복적으로 명시. [Estimate] 태그 적절히 사용

**논리적 비약 1건 지적**:
- 섹션 3.3 해설에서 "누적될수록 더 끈질기게 도전" — 이 해석에 대한 생존자 편향 가능성이 섹션 4.1에서 검토되긴 했으나, 본문(3.3)에서 이 해석을 먼저 단정적으로 서술한 뒤 4.1에서 보완하는 구조는 독자에게 오해를 줄 수 있음. 3.3 시점에서 "다만 이 상승에는 생존자 편향이 기여할 수 있음(4.1절 참고)" 정도의 주석이 있으면 좋겠음

## 확증 편향 체크

### 반증 탐색 충분성: 양호

리포트는 3개의 독립적 반증 탐색을 수행했다:
1. **생존자 편향 검증 (4.1)** — 끈질긴 유저만 남아서 재도전율이 올라간 것인지 검증. 결과: 부분적으로 기여하지만, 개인 수준에서도 간격 단축 확인
2. **사망→이탈 검증 (4.2)** — 사망이 실제 이탈을 유발하는지 확인. 결과: 극소(1~3명)
3. **미클리어자 행동 (4.3)** — 못 깬 유저가 좌절하여 떠난 것인지 확인. 결과: 시도 횟수가 적고 다른 활동 지속

### "찾고 싶은 걸 찾은" 징후 평가

- 초기 가설이 "재도전율 감소"였고 이는 **기각**됨 → 확증 편향의 반대 방향. 가설을 솔직하게 기각한 점은 높이 평가
- 다만 수정된 발견인 "건강한 벽"이라는 결론에 대해서는, 이를 반증하는 시도가 상대적으로 약함. 예를 들어:
  - "생존자 편향의 정확한 기여도"를 분리하지 못했다고 명시했지만, 정성적으로 "복합 효과"라고 결론지은 것은 다소 낙관적 해석일 수 있음
  - 알파 테스터의 선발 효과가 얼마나 강한지에 대한 정량적 검토 없이 "한계"로만 언급

### 기각 조건 공정성

- 초기 가설의 기각 조건이 명확하고, 데이터가 기각 조건을 충족하여 기각됨 — **공정**
- 수정된 발견("건강한 벽")에 대해서는 별도 기각 조건이 설정되지 않음 — 이 점은 개선 가능. "어떤 데이터가 나오면 '좌절의 벽'으로 판정할 것인가"를 사전에 정의했다면 더 엄밀했을 것

| 5.3-3 | "미클리어 유저에 대한 개입은 불필요할 수 있다" | [Estimate] | 적절 |

### 인과관계 과잉 주장 여부

- "사망이 이탈을 유발하지 않음" (요약 4번) — 이 표현은 인과관계 부정으로 읽힐 수 있음. 데이터로 확인된 것은 "사망 이후 이탈한 유저가 극소수"라는 상관관계이며, 인과 부정을 단정하기 어려움. "사망 후 이탈 사례는 극히 드물다" 정도가 더 정확
- "레벨업이 벽을 넘게 함" (요약 5번, 섹션 3.5 제목) — 레벨과 사망률의 상관관계는 확인되었으나, "레벨업이 원인"이라는 인과 방향은 데이터만으로 단정할 수 없음 (높은 레벨 유저가 원래 실력이 좋을 수도 있음). 제목은 "레벨별 사망률 변화" 등 서술적 표현이 더 적절

### 평가

Fact/Estimate 태깅은 결론·시사점에 적절하게 적용되어 있음. 다만 본문 내 **서술과 제목**에서 2건의 과잉 인과 표현이 있어 수정 권고.

## 구조 및 가독성

### 구조 준수 여부

| 필수 섹션 | 포함 여부 | 비고 |
|-----------|----------|------|
| 요약 | O | 5개 항목, 핵심 수치 포함 |
| 연구 배경 | O | 분석팀 기존 발견 + 탐색 동기 |
| 가설 | O | 예상 결과 + 기각 조건 명시 |
| 분석 결과 | O | 6개 하위 섹션 |
| 반증 탐색 | O | 3개 반증 시도 |
| 결론 및 시사점 | O | 가설 기각 + 수정 발견 + 3개 시사점 |
| 한계 및 후속 연구 | O | 5개 한계 + 4개 후속 제안 |
| 부록 | O | 쿼리 로그 + 말할 수 있는 것/없는 것 |

> 구조 완전 준수. 부록 B("말할 수 있는 것 / 말할 수 없는 것 / 필요한 것")는 표준 구조 외 추가 항목으로, 독자에게 매우 유용.

### 가독성

- 비전문가 독해 가능 수준: **양호** — 요약이 명확하고, 해석 A/B 프레임이 직관적
- 전문 용어 병기: **양호** — 생존자 편향(survivorship bias) 등 병기됨
- 차트/시각화: **미포함** — 테이블만 존재. 규정상 "차트/시각화 필수"이므로 추가 필요. 특히 섹션 3.3(재도전율 추세), 3.5(레벨별 사망률 곡선)은 차트로 표현하면 이해도가 크게 향상될 것

### 스토리 흐름

- 섹션 간 연결 자연스러움
- "가설 → 기각 → 수정 발견"의 내러티브가 명확
- 동어반복 최소화

## 수정 권고사항

### 반드시 수정 (수치 오류)

1. **"6회+ 시도자 클리어율 95.9%" → 94.9%로 수정** — 요약 3번, 섹션 3.4, 섹션 5.2 비교표, 부록 B 모두 해당
2. **섹션 3.5 Lv33+ 데이터 보완** — Lv37~46 합계 25건이 누락됨. Lv33+ 합산 시 89건, 사망률 12.4%, 클리어율 70.8%으로 수정. 각주의 레벨별 건수 명세도 갱신 필요
3. **섹션 3.3 원본 쿼리 확인** — 사망 건수 분포(183/145/104/81/62/222)가 재현되지 않음. 원본 쿼리의 streak 정의를 확인하고, 재도전율 수치도 함께 재검증 필요

### 권고 수정 (표현·논리)

4. **"사망이 이탈을 유발하지 않음" → "사망 후 이탈 사례는 극히 드물다"** — 인과 부정 표현을 상관 표현으로 완화
5. **"레벨업이 벽을 넘게 함" → "레벨별 사망률 변화"** — 인과 방향 단정 완화
6. **섹션 3.3 본문에 생존자 편향 참조 추가** — "누적될수록 더 끈질기게 도전" 해석 직후 "다만 이 상승에는 생존자 편향이 부분적으로 기여할 수 있다(4.1절 참조)" 추가
7. **차트/시각화 추가** — 최소 2개(연속 사망별 재도전율 추세, 레벨별 사망률/클리어율 교차 곡선) 권고
8. **섹션 4.2 이탈 기준 명확화** — "12/09 이전"이 `< 12/09`인지 `<= 12/09`인지 명시. 현재 수치(2명)가 어느 기준과도 정확히 일치하지 않으므로 확인 필요

## 종합 판정: MINOR

핵심 결론("dungeon-01은 좌절 요인이 아닌 도전 동기를 제공하는 건강한 벽")을 뒤집는 수준의 오류는 없다. 대부분의 핵심 수치가 재검증에서 일치하며, 가설 정의·기각·반증 탐색의 구조가 엄밀하다.

다만 "6회+ 클리어율 95.9%→94.9%" 산술 오류, Lv33+ 데이터 누락(25건), 섹션 3.3 사망 건수 분포 재현 불가의 3건은 정정이 필요하며, 인과 과잉 표현 2건은 완화를 권고한다.

수정 반영 후 PASS로 재판정 가능.
