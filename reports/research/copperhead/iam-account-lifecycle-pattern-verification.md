## 2. 필수 수정 (2건)

### 수정 1: 양쪽 이중 기록 계정 수 (섹션 3.3)

**현재**: "양쪽 모두 이중 | 1,150"
**검증**: 이중 등록 1,151개 계정이 **전원** 이중 로그인에도 해당 → 교집합 = **1,151**
**영향**: 수치 1건 오기. "1,150 → 1,151" 수정 필요.

**영향**: 경미하지만 산술 정확성 기준으로 수정. "288%" → "289%" 또는 "약 289%"

## 3. 경미 수정 (2건)

### 경미 1: 섹션 3.2 "44,551" 설명

**현재**: "44,551은 gpp_id의 MIN(device_id) 기준 조인 결과로, 44,549와 2건 차이는 쿼리 집계 시 동시 발생 이벤트의 처리 차이임"
**검증**: 모든 계정이 기기 1대에서만 등록하므로, MIN(device_id) 기준이든 DISTINCT 기준이든 결과가 동일해야 한다. 실제로 현재 inner join 결과는 44,557(= 고유 계정 수)이다.
**제안**: 리포트 작성 시점의 쿼리가 다른 조건을 포함했을 수 있으나, "2건 차이" 설명이 검증 불가. 해당 괄호 설명을 삭제하고, 기기 일치율을 "100% (N=44,549)" 기준으로 통일하거나, "44,549쌍 전원 기기 일치"로 단순화 권고.

### 경미 2: 이중 기록 규모 표 해석

**현재**: "등록 이중 기록(1,151건)과 로그인 이중 기록(1,510건)의 차이(359건)는..."
**검증**: 로그인 이중 기록은 현재 1,512건(리포트 시점 1,510, 데이터 성장 +2). 차이 = 1,512 - 1,151 = 361건. 리포트 시점에도 1,510 - 1,151 = 359이므로 산술 자체는 맞지만, 설명 중 "3회 이상 로그인 2건"과 "is_first_login 도입~이중기록 사이 재로그인"이 359건을 구성한다는 설명은 충분히 검증하기 어렵다. 다만 핵심 결론에 영향이 없으므로 경미 사항으로 분류.

## 7. 종합 평가
