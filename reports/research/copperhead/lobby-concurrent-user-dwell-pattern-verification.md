## 1. 검증 요약

21개 핵심 수치를 독립 쿼리로 재실행한 결과, **15개 완전 일치, 4개 미세 오차(1~2건 경계 차이), 2건 수정 필요**. 논리 구조와 가설 판정은 전반적으로 타당하다. 반증 탐색 3건이 수행되었고 heartbeat 교차 검증까지 포함하여 근거의 다원화가 양호하다.

수정이 필요한 항목:
1. **자동화 비율 표기 혼선** (connected/disconnected 테이블 혼용)
2. **금요일 02-13 최대 CCU**: hour 21에서 42, hour 22에서 44 -- 테이블과 본문에서 44로 표기하였으나 hour 21 기준이라면 42가 정확
3. **섹션 4.2 체류 시간 분포 테이블의 구간 누락** (30~60초 421건이 빠져 합계 6,991건)
4. **"9주 중 8주" vs "10주 중 9주" 불일치**
5. **sessionstart 매칭 수치 불일치** (345 vs 검증값 310)

### 2.3 수정 필요 항목 (5개)

#### (A) 자동화 비율 표기 혼선 (섹션 2, 데이터 탐색)

리포트 기술:
> "로비 접속 16,004건 중 8,480건(53.0%)이 자동화 기기에서 발생"

검증 결과:
- `view_lobby_connected`: 16,004건 (auto 8,480, non-auto 7,524) -- **8,480/16,004 = 53.0%** 정확
- `view_lobby_disconnected`: 15,792건 (auto 8,380, non-auto 7,412)
- 이후 본문에서 "자동화 (8,380건)"이라 표기하는데 이는 disconnected 기준

**문제**: connected 기준 총량(16,004)과 disconnected 기준 자동화 수(8,380)를 혼용. 섹션 2에서는 8,480이 맞고, 섹션 4.2 비교 테이블의 "자동화 (8,380건)"은 disconnected 기준이므로 둘 다 정확하지만, 독자가 8,480과 8,380의 차이(100건)를 혼동할 수 있음.
**권고**: 섹션 2에서 "로비 접속(connected) 16,004건 중 8,480건(53.0%)"으로 테이블명 명시, 또는 disconnected 기준으로 통일.

#### (B) 금요일 02-13 최대 CCU 오기 (섹션 4.1, 주차별 테이블)

리포트 기술:
> | 02-13 | 20.4 | **44** |

검증 결과:
- 02-13 금요일 UTC **21시**: 평균 20.4, 최대 **42**
- 02-13 금요일 UTC **22시**: 평균 14.7, 최대 **44**

**문제**: 테이블 제목은 "금요일 UTC 21시 CCU 추이"이므로 hour=21 기준이어야 하는데, 02-13의 최대 CCU 44는 hour=22에서 발생. hour=21 기준 최대는 42.
**영향**: 전체 최대 CCU(44)는 변하지 않으나, "금요일 UTC 21시"라는 필터 조건과 테이블 내 수치가 불일치.
**권고**: 02-13 행의 최대 CCU를 42로 수정하거나, 테이블 범위를 "UTC 21~22시"로 확장.

#### (C) "9주 중 8주" 오기 (섹션 4.1, 본문)

리포트 기술 (줄 119):
> "9주 중 8주에서 금요일 UTC 21시 최대 CCU가 22 이상"

리포트 기술 (줄 121):
> "10주 중 9주에서 반복됨"

검증 결과:
- 기간 내 금요일 10주 (02-06 ~ 04-10)
- 최대 CCU >= 22인 주: 9주 (03-06만 9로 예외)

**문제**: 줄 119의 "9주 중 8주"는 오기. 줄 121의 "10주 중 9주"가 정확.
**권고**: 줄 119를 "10주 중 9주"로 수정.

#### (D) 섹션 4.2 체류 시간 분포 테이블의 구간 누락

리포트 테이블:

| 구간 | 세션 수 |
|------|--------|
| 0분 미만(30초 이내) | 1,479 |
| 1~2분 | 775 |
| 2~10분 | 1,818 |
| 10~42분 | 1,377 |
| 42~53분 | 1,441 |
| 53분 초과 | 101 |

합계: 1,479 + 775 + 1,818 + 1,377 + 1,441 + 101 = **6,991**

검증 결과:
- 전체 비자동화 세션: **7,412건**
- 누락 구간: 30~60초 = **421건**
- "0분 미만(30초 이내)"는 duration < 30 (1,479건)
- "1~2분"은 duration 60~119초 (775건)
- 30~60초 421건이 어떤 구간에도 포함되지 않음

**문제**: 비율 열의 분모(7,412)는 맞지만 행 합산이 7,412와 421건 차이. 독자가 행을 합산하면 불일치를 발견함.
**영향**: 결론에는 영향 없으나 데이터 정합성 신뢰도에 영향.
**권고**: "0분 미만(30초 이내)" 행을 "30초 미만"으로 명확히 하고, "30초~1분" 행(421건, 5.7%)을 추가하거나 "1~2분" 행의 범위를 "30초~2분(1,196건, 16.1%)"으로 변경하여 구간을 연속으로 만들기. (참고: 섹션 4.3 미션 참여 테이블에서는 "30초~2분 = 1,196"으로 올바르게 처리됨)

#### (E) sessionstart 매칭 수치 불일치 (섹션 4.4)

리포트 기술:
> "이 중 345명(26.4%, 345/1,306)은 sessionstart 테이블에 기록이 있어"

검증 결과:

- `view_lobby_disconnected.krafton_id = sessionstart.krafton_id` 매칭: **310명 (23.7%)**
- 날짜 필터 제거 후에도 동일: **310명**

**문제**: 35명(345 - 310) 차이의 원인을 특정하지 못했으나, 3가지 조인 방식 모두 310으로 수렴.
**영향**: "73.6%가 게임 세션도 생성하지 않았다"는 결론의 방향은 유지되나(76.3%로 상향), 수치 정확성 수정 필요.
**권고**: 310명(23.7%)으로 수정. 나머지 996명(76.3%)으로 수정.

## 3. 논리 검증 결과

### 3.1 H1: 금요일 피크 (채택)

**기각 조건**: "금요일 피크가 다른 요일 피크와 유의미하게 다르지 않거나, 급등 패턴이 일회성인 경우"

검증 결과:
- 금요일 21시 평균 CCU: 5.7~20.4 (10주)
- 비금요일 최대 CCU: 17 (수요일)
- 금요일 최대 CCU: 22~44 (9주), 9 (1주)

**판정 검토**: 채택 타당. 10주 중 9주에서 패턴 반복. 기각 조건("일회성")에 해당하지 않음. 다만 원래 가설의 "5배 이상"은 2월(5~7배)에만 해당하고 3~4월은 2~4배. 리포트가 이 점을 "감소 추세"로 솔직히 보고한 것은 적절.

**주의**: "5배 이상"이라는 원래 기준을 엄격 적용하면 부분 채택이 더 정확하나, 리포트가 "2~7배 수준"으로 범위를 명시하고 감소 추세를 기술하였으므로 표현 수준의 문제.

### 3.2 H2: 이중 분포 (채택)

**기각 조건**: "분포가 단봉 또는 균일에 가까운 경우"

검증 결과:
- ROUND 기준 히스토그램: 0분=1,479, 1분=834, ... 중간 감소 ... 43분=155, 45분=159
- 두 피크 사이(1~42분) 분당 24~173건 vs 피크(43~45분 131~159건)
- heartbeat 교차 검증에서도 동일 패턴 (1,347/5,647 = 23.8%)
- 자동화 세션에서는 다른 패턴 (7-8분, 25-26분)

**판정 검토**: 채택 타당. 두 개의 모드가 분명하고, 독립 데이터 소스(heartbeat)에서 교차 확인됨.

### 3.3 H3: 장기 체류 미션 참여율 (부분 채택)

**기각 조건**: "42~53분 세션의 미션 참여율이 다른 구간과 동일하거나 더 높은 경우"

검증 결과:
- 42~53분 미션 참여율: 19.4% (280/1,441)
- 전체 평균: 18.7% (1,384/7,412)
- 2~10분 구간: 29.0%

**판정 검토**: "부분 채택" 적절. 42~53분 구간이 2~10분 대비 낮지만(9.6%p 절대차), 전체 평균과는 차이 없음. "현저히 낮다"는 기각되었고, "평균 수준이다"가 결론. 다만 미션 참여 판정이 사용자 수준(세션 수준 아님)이라는 한계를 리포트가 명확히 기술한 것은 양호.

### 3.4 [Estimate] 타임아웃 추정 (섹션 6.3)

리포트 추정: "로비 서버 측 세션 타임아웃(약 45~50분 설정)"

검증:
- 53분 초과 세션: 100건 (1.4%), 42~53분: 1,441건 (19.4%)
- heartbeat에서도 동일 패턴
- 자동화 세션에서는 미발생
- [Estimate] 태그 부착, 서버 설정 미확인 명시

**판정**: 추정의 근거(극소수 53분+ 세션, heartbeat 교차 확인, 자동화 미발생)가 충분하고, [Estimate] 태그와 단정 불가 단서를 포함. 적절한 수준의 추론.

- 리포트 본문에서 "존재를 강하게 시사한다"로 완화하였으나, 소제목이 단정적
- **권고**: 소제목을 "금요일 정기적 피크가 CCU의 결정적 동인"으로 수정하거나, [Estimate] 태그 부착

**42~53분 = 타임아웃 해석**:
- [Estimate] 태그가 적절히 부착됨
- 대안 해석(플레이 사이클, 클라이언트 측 요인)도 언급
- **양호**

**전체 평가**: 확증 편향의 위험이 낮은 편. 가설 기각/부분 채택을 솔직히 보고하였고, 반증 탐색이 3건으로 충분. 소제목의 단정적 표현 1건만 수정 권고.
