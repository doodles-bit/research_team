## 1. 교차 분석 발견

### 1.1 토큰 유효기간 ~48분 = 로비 체류 42~53분 타임아웃? (#14 + #15)

| 연구 | 수치 |
|------|------|
| #14 로비 CCU | 체류 시간 42~53분에 19.4%(1,441건) 집중. 53분 초과 세션 1.4%(101건)에 불과 |
| #15 IAM | 등록→첫 토큰 갱신까지 평균 2,879초(~48분). 갱신 빈도와 무관하게 일정 |

**교차 인사이트**: 두 독립 리포트가 다른 테이블에서 같은 ~48분 상한을 측정했다. 로비의 42~53분 "타임아웃" 패턴은 서버 측 세션 타임아웃이 아니라 **IAM 토큰 만료(~48분)에 의한 자연 세션 종료**일 가능성이 높다. 토큰 갱신이 발생한 계정(2.9%)만 48분을 넘길 수 있고, 갱신이 없으면 토큰 만료로 세션이 종료되는 구조다.

**보강 근거**:
- 자동화 기기에서 토큰 갱신 0건(#15) + 자동화 세션 중앙값 17.6분(#11) → 자동화는 48분 이내에 끝나므로 토큰 갱신 불필요
- 토큰 갱신 계정의 로비 접속 4.08회 vs 비갱신 1.05회(#15) → 갱신 계정만 장시간 활동 가능
- heartbeat 데이터에서도 동일한 42~53분 집중 관측(#14) → 두 독립 소스에서 교차 확인

**실무적 의미**: 로비 체류 42~53분 종료가 의도된 설계(토큰 만료 = 세션 정리)인지, 장시간 플레이어에게 UX 문제를 일으키는 비의도적 종료인지 확인 필요. 토큰 갱신이 자동으로 이루어지지 않는다면, 긴 세션을 유지하려는 플레이어가 강제 종료되는 상황이 발생할 수 있다.

> [Estimate] 토큰 만료→세션 종료 인과관계는 추정이다. 서버 측 타임아웃 설정이 별도로 존재할 수도 있으며, 두 현상이 우연히 유사한 시간대에 발생하는 것일 수도 있다. 확인하려면 서버 설정 또는 disconnection reason 필드가 필요하다.

### 1.2 "로비 이탈" — 3개 리포트가 다른 계층에서 측정한 동일 현상 (#13 + #14 + #15)

| 연구 | 테이블 | 로비 이탈 수치 |
|------|--------|-------------|
| #13 맵 스폰 | cphplayerspawned | 37.4%(671/1,795명) 로비-only — 미션 맵 미도달 |
| #14 로비 CCU | view_lobby_disconnected | 20.0%(1,479건) 30초 이내 즉시 이탈 |
| #15 IAM | IAM→lobby→session 퍼널 | 27.9%(12,421명) 로비→세션 미전환 |

**교차 인사이트**: 세 리포트가 서로 다른 테이블과 모수(母數)에서 "로비에서 게임 본편으로 넘어가지 못하는 비율"을 측정했다. 20~37% 범위로 수렴하며, 이는 Copperhead 베타의 핵심 퍼널 병목이 로비→게임 전환 단계임을 다각도로 확인한 것이다.

**수치 차이의 구조적 원인**:
- #13(37.4%)은 스폰 기록이 있는 1,795명 한정(커버리지 40.1%)으로 과대 추정 가능
- #14(20.0%)은 30초 이내 이탈만 측정 — 30초~수분 이탈 미포함
- #15(27.9%)은 IAM 전체 44,549 계정 기준으로 가장 큰 모수 — 대표성 최고

**[추적 대상]**: 로비 이탈의 원인 구분(기술적 실패 vs 의도적 이탈 vs 자동화 패턴)이 아직 미해결. 향후 disconnection reason 또는 로비 체류 시간과 미션 참여의 세션 단위 매칭으로 해결 가능.

### 1.3 자동화 테스트의 전 계층 침투 — 종합 정리 (#11 + #14 + #15)

| 계층 | 테이블 | 자동화 비율 |
|------|--------|-----------|
| PC 환경/세션 | sessionstart | 60.3%(6,805/11,285명) |
| 로비 접속 | view_lobby_connected | 53.0%(8,480/16,004건) |
| IAM 등록 | view_iam_registered | 54.3%(24,795/45,700행) |

**교차 인사이트**: 자동화 테스트가 IAM(등록)→로비(접속)→게임(세션) 전 계층에서 50~60% 비중을 차지한다. 다만 비율이 계층마다 다른 이유가 있다:
- IAM(54.3%): 등록 행 기준. 자동화 기기 1대가 생성하는 일회용 계정이 많으나, 비자동화 기기도 263대로 다양
- 로비(53.0%): 건 기준. 자동화 기기 4대(상위)가 1,393~2,222명의 계정을 운용하여 건수 지배
- 세션(60.3%): 명 기준. 자동화 기기 6대가 800~1,395명 운용

추가로, #15에서 발견된 **비공식 대량 계정 기기 5대**(비자동화 중 1,000+계정)가 공식 7대에 포함되지 않은 자동화 환경일 가능성이 제기됨. 이를 포함하면 실제 자동화 비율은 60%를 상회할 수 있다.

**실무적 의미**: 모든 Copperhead 분석에서 자동화 필터링은 필수이며, 공식 7대 외 비공식 자동화 기기의 식별이 데이터 정확성에 중요하다.

**추세**: 오류 유형은 매 연구마다 다르지만(8~9종류가 모두 다른 유형), **심각도는 점진적으로 감소**하고 있다. #15는 코퍼헤드R의 전체 9건 연구 중 수치 정확도가 가장 높다(검증원 평가). 셀프 체크리스트(7항목) 도입 후 수정량이 줄어드는 추세.

## 6. 종합 — Copperhead 9건 연구를 통해 그려진 전체 그림

15건의 연구(PalM 6건 + Copperhead 9건)를 완료하면서, Copperhead 베타의 전체 사용자 여정이 연결되었다:

```
[IAM 등록] 100% headless, 일회용 계정 생성 (#15)
    ↓ 1초 이내
[IAM 로그인] 100% 전환, 기기 기반 자동 인증 (#15)
    ↓ 98.8% 도달
[로비 접속] 금요일 UTC 21시 피크, 이중 체류 분포 (#14)
    ↓ 72.1% 도달 (27.9% 이탈 — 핵심 병목)
[게임 세션] 60.3% 자동화, 77.2% 솔로, 계정-기기 1:1 (#11, 3차)
    ↓
[미션 진행] 허브-스포크 구조, 37.4% 로비-only (#13)
    ↓
[전투] AR/Sword/DMR 3강, HitReact 근접 취약 (#12, 4차)
    ↓
[미션 결과] 80.7% 텔레메트리 누락, Defend 9.1% (#7)
    ↓ ~48분 토큰 만료
[세션 종료] 42~53분 체류 집중, 88.7% 재방문 없음 (#14, #15)
```

이 여정에서 가장 큰 레버리지 포인트는:
1. **로비→세션 전환(27.9% 이탈)**: 가장 큰 단일 이탈 구간
2. **토큰/세션 상한(~48분)**: 장시간 플레이 의지가 있어도 강제 종료 가능성
3. **자동화 50~60% 비중**: 모든 수치 해석에 필터링 필수
4. **텔레메트리 누락(80.7%)**: 미션 레벨 분석의 근본적 제약
